# OmniCore

OmniCore is a collaborative intelligence project built around one idea: human vision and AI execution should work together to turn ideas into real systems.

This repository is the working space for that process. It currently includes:

- a browser-based Snake prototype in `index.html`, `styles.css`, and `src/`
- a Roblox sync setup in `roblox/` powered by Rojo
- the OmniCore foundation documents that define how strategy and execution should operate together
- Alpha's personal vision document that defines the long-term direction the system should support

## Core idea

OmniCore combines three roles:

- Alpha: vision, direction, and high-level purpose
- Strategy Intelligence: reasoning, system design, and planning
- Execution Intelligence: implementation in code, tools, and workflows

The goal is not passive assistance. The goal is collaborative building.

## Working principles

OmniCore follows a simple operating loop:

1. Think first
2. Analyze second
3. Build third
4. Improve continuously

This repository should favor:

- simplicity over unnecessary complexity
- modular systems over tightly coupled ones
- clarity over cleverness
- scalable foundations over short-term hacks

## Repository structure

- `OMNICORE_GENESIS.md`: source philosophy and identity document
- `ALPHA_VISION.md`: Alpha's long-term goals, values, and direction
- `OMNICORE_ROADMAP.md`: product direction, MVP scope, and build phases
- `AGENTS.md`: operating instructions for agents and collaborators in this repo
- `src/`: browser game logic
- `roblox/src/`: Roblox code synced through Rojo
- `ROBLOX_SETUP.md`: local setup instructions for Roblox Studio sync

## Strategic filter

When deciding what to build next, use these two documents together:

- `OMNICORE_GENESIS.md` defines how OmniCore should think and operate
- `ALPHA_VISION.md` defines what Alpha is ultimately trying to achieve

A good project in this repository should increase one or more of these:

- freedom
- capability
- income potential
- automation leverage
- long-term system value

Current short-term priority:

- build toward at least 500 EUR in consistent monthly online income
- favor projects that can be run remotely
- use income as a bridge to full-time OmniCore development

## How to work in this repo

When adding features or starting a new direction:

1. Clarify the real objective
2. Surface assumptions and risks
3. Look for the simplest viable architecture
4. Implement in small, testable steps
5. Leave the system easier to evolve than before

## Local development

Run the project locally with:

```powershell
npm run dev
```

Sanity-check the app logic with:

```powershell
npm run check
```

## Deployment

This repository includes a GitHub Pages workflow at `.github/workflows/pages.yml`.

Once the repo is pushed to GitHub on the `main` branch and Pages is enabled for GitHub Actions, the current static OmniCore app can be deployed automatically.

## Near-term direction

The current codebase is still an early sandbox. Good next steps include:

- shaping a clearer OmniCore product direction around income and leverage
- expanding the web prototype into something closer to a real product surface
- building reusable modules for future tools, automations, and online services
- prioritizing experiments with realistic monetization paths
- growing the Roblox side only if it supports skill growth, distribution, or income opportunity

The current recommended direction is to turn the existing web prototype into the first OmniCore MVP workspace.

## Long-term direction

OmniCore is intended to grow into a broader ecosystem that may include:

- automation systems
- personal AI operating workflows
- development tooling
- intelligent assistants for design and execution
- robotics or physical-system integrations

Start simple. Build deliberately. Improve continuously.
